# UX Playbook condensé

## Patterns obligatoires
- états vides pédagogiques
- feedback immédiat
- erreur inline sur formulaires
- autosave ou protection contre perte
- undo pour actions courantes
- confirmation seulement pour l’irréversible
- recherche globale rapide
- tables avec filtres, tri, bulk actions
- responsive adaptatif
- accessibilité clavier / contraste / labels

## Anti-patterns
- cloner le desktop sur mobile
- multiplier les confirmations inutiles
- ajouter des animations qui ralentissent
- sacrifier la compréhension pour l’esthétique
- dépendre d’une seule couleur pour transmettre une information
